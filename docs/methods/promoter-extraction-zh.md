**核心原则：启动子位于基因转录起始位点（TSS）的上游。**

对于正链（+）基因，上游就是坐标更小的方向；对于负链（-）基因，上游则是坐标更大的方向。

用你给的四个基因来具体分析：

**AT1G01010**（+链，3631–5899）：TSS在3631。它是chr1的第一个基因，上游没有其他基因，所以启动子区域可以取 1–3630。你说的"到360"应该是3630，这个没问题。

**AT1G01020**（-链，6788–9130）：因为是负链基因，TSS在9130（坐标最大的那端）。上游方向是坐标增大的方向，所以启动子区域是 9131 到下一个基因（AT1G03987，起始于11101）之前，即 9131–11100。这个推断也是合理的。

**AT1G03987**（+链，lncRNA，11101–11372）：TSS在11101，上游就是AT1G01020的右端（9130）之后，即 9131–11100。你会发现这段区域和AT1G01020的推断启动子重叠了——这是这种方法的一个天然问题：**当两个基因"背靠背"（head-to-head）排列时，它们会共享同一段基因间区作为各自的启动子。**

**AT1G01030**（-链，11649–13714）：TSS在13714，上游是坐标更大的方向，需要看再下一个基因的位置来确定边界。

关于这个方法，有几点值得考虑：

第一，实际操作中，很多人不会取整个基因间区，而是设一个**固定窗口**，比如 TSS 上游 1kb、1.5kb 或 2kb。因为真正的核心启动子（core promoter）通常就在 TSS 上游几百 bp 以内，而调控元件（regulatory elements）一般在 2–3kb 范围内。取整个基因间区的话，如果两个基因间距很大（比如几十kb），那"启动子"就太长了，引入很多噪声。

第二，你用的是基因（gene feature）的坐标边界作为 TSS，这其实是 mRNA/转录本的最外端坐标（包含 UTR）。对于有多个转录本的基因（比如 AT1G01020 有多个 mRNA），不同转录本的 5'UTR 末端可能不同，TSS 也会不同。更精确的做法是**取每个转录本的 5'UTR 最远端**作为 TSS，或者直接用 CAGE-seq 等实验数据来确定 TSS。

第三，对于负链基因，要特别注意方向别搞反。一个简单的规则：**+链基因的启动子在 start 坐标的左边，-链基因的启动子在 end 坐标的右边。**

总的来说，你的方法作为一种"基因间区即启动子"的粗略估计是完全可用的，尤其适合做全基因组层面的批量分析。如果你需要更精确的启动子注释，可以考虑结合 ATAC-seq、ChIP-seq（H3K4me3）或 CAGE-seq 数据来校准。
