## What this is

Five hand-written, step-by-step bioinformatics audits of the **pre-monorepo** PMET pipelines. ~2700 lines of `command → purpose → biological meaning → input sample → output sample → expected properties → observed result → PASS/FAIL` written when the pipelines lived under `scripts/pipeline/0X_*.sh`.

Kept here as **conceptual reference**: they show the kind of depth one **could** apply when reasoning about whether a pipeline output is biologically and structurally what it should be — coordinate traces for specific genes, reverse-complement checks on `−` strand promoters, IC computation per motif, binomial threshold derivations.

## Important — these docs do **not** auto-update

Unlike the [`docs/workflows/*.md`](../) sibling docs (which are regenerated by `make test-audit` from real runs against canonical inputs), the files in this directory are a **frozen snapshot**. Path references (`scripts/pipeline/03_promoter.sh`, `data/TAIR10.fasta`, etc.) are pre-monorepo and **stale**:

| In these docs | Today's equivalent |
|---|---|
| `scripts/pipeline/03_promoter.sh` | `scripts/workflows/promoter.sh` |
| `scripts/pipeline/04_intervals.sh` | `scripts/workflows/intervals.sh` |
| `scripts/pipeline/05_promoter_gap.sh` | `scripts/workflows/cli/05_promoter_gap.sh` |
| `scripts/pipeline/06_elements_longest.sh`, `07_elements_merged.sh` | one consolidated `scripts/workflows/elements.sh -s longest\|merged` |
| `data/TAIR10.fasta`, `data/TAIR10.gff3` | `data/reference/TAIR10.{fasta,gff3}` |
| `data/Franco-Zorrilla_et_al_2014.meme` | `data/motifs/Franco-Zorrilla_et_al_2014.meme` |
| `results/03_promoter/` | `results/cli/promoter/` |

The **algorithms and biology** described still apply (PMET's two-stage indexing-then-pairing pipeline hasn't changed). It's the **paths and script numbers** that drifted.

## Why we keep them

Three uses:

1. **Pedagogical** — when a new contributor wants to understand "what does step 5 of the promoter pipeline actually compute, and how would I sanity-check its output by hand", these docs show worked examples (specific gene coordinates, reverse-complement verification, etc.) that the auto-generated [`docs/workflows/promoter.md`](../promoter.md) doesn't go into.
2. **Audit pattern** — if we ever extend the [`tests/audit/`](../../../tests/audit/) framework to do per-step traces (not just the end-of-pipeline `motif_output.txt` row that's currently shown as "Worked example"), these are the depth-of-detail target.
3. **Migration provenance** — they record what the pipelines looked like immediately before the monorepo merge, useful when bisecting across the merge.

## 这是什么

五份手写的、按 step 走的 PMET pipeline 生信审计文档，写于 **monorepo 合并之前**。~2700 行，每步按 `命令 → 目的 → 生物学含义 → 输入截样 → 输出截样 → 期望属性 → 观察结果 → PASS/FAIL` 的格式展开，针对当时还在 `scripts/pipeline/0X_*.sh` 路径下的那批脚本。

留在这里作**概念参考**：示范了"如果想推理一条 pipeline 的输出在生物学上和结构上是不是它该是的样子，可以做到多深" —— 具体基因的坐标追踪、`−` 链启动子的反向互补校验、per-motif IC 计算、binomial 阈值推导。

## 重要 —— 这些文档**不会**自动更新

跟兄弟目录 [`docs/workflows/*.md`](../)（由 `make test-audit` 在 canonical 输入上真实跑出来再重生成）不同，本目录里的文件是**冻结快照**。路径引用（`scripts/pipeline/03_promoter.sh`、`data/TAIR10.fasta` 等）是 monorepo 之前的、**已过期**：

| 文档里写的 | 今天的对应 |
|---|---|
| `scripts/pipeline/03_promoter.sh` | `scripts/workflows/promoter.sh` |
| `scripts/pipeline/04_intervals.sh` | `scripts/workflows/intervals.sh` |
| `scripts/pipeline/05_promoter_gap.sh` | `scripts/workflows/cli/05_promoter_gap.sh` |
| `scripts/pipeline/06_elements_longest.sh`、`07_elements_merged.sh` | 合成一份 `scripts/workflows/elements.sh -s longest\|merged` |
| `data/TAIR10.fasta`、`data/TAIR10.gff3` | `data/reference/TAIR10.{fasta,gff3}` |
| `data/Franco-Zorrilla_et_al_2014.meme` | `data/motifs/Franco-Zorrilla_et_al_2014.meme` |
| `results/03_promoter/` | `results/cli/promoter/` |

讲的**算法和生物学**仍然成立（PMET 的两阶段 indexing → pairing 没变）。漂移的是**路径和脚本编号**。

## 为什么留着

三种用法：

1. **教学** —— 新人想理解"promoter pipeline 第 5 步到底算什么、我怎么用手工验证它的输出"时，这些文档给出了具体例子（某基因的坐标、`−` 链反向互补验证等），自动生成的 [`docs/workflows/promoter.md`](../promoter.md) 没这么深。
2. **审计模式参照** —— 如果未来把 [`tests/audit/`](../../../tests/audit/) 框架扩展到 per-step 追踪（不仅是当前结尾的"Worked example"那一行 motif_output），这些文档是深度目标。
3. **迁移溯源** —— 记录了 monorepo 合并前一刻 pipeline 的样子，跨合并点 bisect 时有用。
